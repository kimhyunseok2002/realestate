# 살아남을지도 · SurviMap

> **입지·업종별 상권 생존 예측 서비스** — 위치 + 업종을 넣으면, 그 자리에서 그 업종이 1년 / 3년 / 5년 뒤 살아남을 확률을 예측하고, 무엇이 생존율을 깎고 받쳐주는지 분해해 자연어 리포트로 돌려줍니다.
>
> 브랜드명 **살아남을지도** = "(장사가) 살아남을지도" + "살아남을 지도(地圖)" — 생존 예측과 지도 기반을 한 단어에 담은 중의적 워드마크. 영문 **SurviMap** (Survive + Map). 팀: 김현석 · 최지훈.

기획 문서의 핵심 구조 — **"ML 예측 엔진 + LLM 인터페이스"** — 를 그대로 구현한 작동 프로토타입입니다.

- **예측 그 자체는 ML 코어**(생존분석 Cox·Kaplan-Meier + Weibull)가 담당합니다.
- **LLM(Claude)은 예측 엔진이 아니라** 그 숫자를 소상공인이 읽는 자연어로 **번역·설명하는 레이어**입니다. (판단 대체 ✗ / 검증 지원 ✓)

## 빠른 실행

```bash
cd ~/Desktop/상권생존예측
python3 run.py
# 또는  ./run.sh
```

브라우저에서 **http://127.0.0.1:8000** 접속 → 강남역·카페로 자동 예측된 대시보드가 뜹니다.
지도 위 **자치구 생존율 버블**을 누르거나, 지도를 클릭하거나, 주소·역·상호를 검색하세요 (입력 중 실시간 검색 · ↑↓/Enter 키보드 탐색).

> 포트 변경: `PORT=8080 python3 run.py`

### 필요 조건
- Python 3.10+ (numpy 시스템 설치 가정)
- `fastapi`·`uvicorn` 은 `backend/vendor/` 에 이미 포함되어 별도 설치 불필요
- 인터넷 (지도 타일·주소검색·CDN 라이브러리·OSM 실측)
- **Claude CLI** (있으면 실제 LLM 리포트, 없으면 자동으로 템플릿 리포트로 폴백 — 서비스는 항상 동작)

### Vercel 배포
`api/index.py`(ASGI 엔트리) + `vercel.json` 이 포함되어 있어 GitHub 연동만으로 서버리스 배포됩니다.
Vercel에서는 Claude CLI가 없으므로 AI 총평은 템플릿으로 폴백하며, 심층 리포트(결정론적 계산)는 그대로 동작합니다.

### 심층 리포트 (유료 · 맥킨지급 7섹션)
결제(데모) 또는 구독 플랜으로 열람하며, 내 조건(면적·월세·보증금·권리금·자기자본·목표 월순익)을 입력해 재계산할 수 있습니다.

1. **손익분기·투자회수 시뮬** — 월 P&L, 손익분기 매출, 낙관/기본/최악 시나리오, 초기투자 분해, 회수기간, 목표 대비
2. **실행 플레이북** — 받쳐주는 강점 / 개선해야 살아남는 요인별 구체 전략
3. **실패 시나리오 3가지** — 경쟁 진입·임대료 인상·배후 위축을 엔진으로 재계산한 수치 + 대비책
4. **적정 임대료 분석** — 적정 임대료 vs 제시액, 협상 목표가
5. **상권 추세** — 유동인구·공실률 24개월 추이 (뜨는지/지는지)
6. **더 나은 대안** — 예산 안 더 안전한 자리 + 이 자리에 더 맞는 업종
7. **실측 코호트 근거** — 유사 점포 생존 이력(KM) + 반경 내 경쟁 밀도

TXT 저장 · **인쇄용 PDF**(리포트 전용 5페이지, 배경색 보존) 지원.

## ML 예측 엔진 방법론

`backend/survival.py` 는 교과서적 생존분석을 그대로 구현합니다.

1. **Cox 비례위험** — 표준화된 상권 변수 `z` 에 업종별 회귀계수 `β` 를 곱해 선형예측자 `LP = Σβⱼzⱼ`, 위험비 `HR = exp(LP)`.
2. **Weibull 기저 생존함수** — `S₀(t) = exp(-(t/scale)^shape)`. `shape<1` 이 외식업의 초기 폐업위험이 큰 '죽음의 계곡'을 표현.
3. **점포 생존함수** — `S(t) = S₀(t)^HR`.
4. **Kaplan-Meier** — 같은 위험모형과 정합한 **모형 추정 코호트**를 만들어 비모수 생존곡선을 산출(실 폐업 이력 연결 전까지는 모형 기반 참고치).
5. **위험 분해** — 각 변수의 LP 기여분을 3년 생존율 변화(%p)로 환산.
6. **what-if 시뮬** — 임대료·유동인구·경쟁 밀도를 비율로 조정해 z-재계산 → 생존율 실시간 재산출.

`backend/report.py` 는 그 위에 **재무 계산층**을 얹습니다 — 업종별 원가 구조(원재료·인건비·기타), 평당 예상매출, 손익분기, 초기투자(보증금·권리금·인테리어·운영자금), 회수기간, 임대료 적정가.

## 실데이터 연결 현황 (hybrid)

임대료·공실률은 **실측값**으로 연결됐고, 나머지 상권 변수는 아직 합성값입니다.
스키마(필드 이름)는 그대로 두고 값만 채우는 설계라, 소스별로 하나씩 실데이터로 교체할 수 있습니다.

| 역할 | 데이터 | 출처 | 상태 |
| --- | --- | --- | --- |
| **임대료·공실률** | **상업용부동산 임대동향(소규모 상가, 분기)** | **한국부동산원 R-ONE OpenAPI** | ✅ **연결됨(2024 2분기, 전국 상권 실측)** |
| 교통·앵커 | 지하철·버스·주차 / 대학·병원·마트 등 | OpenStreetMap | ✅ 연결됨 |
| 라벨(생존기간) | 인허가·개폐업 데이터 | 공공데이터포털 / 구 LOCALDATA | ⏳ 예정 |
| 점포 변수 | 상가정보(업종·좌표·경쟁밀도) | 공공데이터포털 | ⏳ 예정 |
| 상권 변수 | 매출·유동인구 | 서울 우리마을가게 상권분석 | ⏳ 예정 |
| 배후 변수 | 인구·사업체 일자리 | 통계청 SGIS | ⏳ 예정 |
| 매물 | 월세·보증금·면적·층 | 실 매물 API 확보 시 `listings.py` 교체 | ⏳ 예정 |

**R-ONE 실데이터 파이프라인** (`backend/realdata/`):
`build_rone.py` 가 R-ONE OpenAPI에서 소규모 상가 임대료(천원/㎡)·공실률(%)을 분기별로 받아,
`rone_crosswalk.py`(R-ONE 주요상권 → 자치구 매핑)로 자치구 대표값을 계산해
`rone_lease.json`(공개 통계 스냅샷)으로 저장합니다. 서비스 구동 시 `loader.py` 가 이 스냅샷을
`DISTRICTS` 에 주입합니다(자치구 81개 중 69개는 상권 실측, 12개는 시·도 평균 폴백).
새 분기 갱신: `RONE_API_KEY=... python3 -m backend.realdata.build_rone`

> 문서의 제언대로 **서울 특정 자치구 + 1~2개 업종**부터 실데이터로 검증한 뒤 확장하는 것이 안전합니다.
> 미지원 지점은 가장 가까운 지원 상권으로 스냅해 근사 분석합니다.

## LLM 설정 (OpenAI API)

- 백엔드가 **OpenAI Chat Completions API** 를 HTTP(urllib)로 직접 호출합니다. SDK 의존성이 없어 Vercel 서버리스에서도 그대로 동작합니다.
- 키는 환경변수 **`OPENAI_API_KEY`** 로만 읽습니다(코드에 넣지 않음). 키가 없거나 호출 실패/타임아웃이면 **규칙 기반 템플릿**으로 자동 폴백 → LLM 없이도 서비스는 항상 동작.
- 기본 모델은 **`gpt-5.5`**(추론모델). GPT-5 계열은 `max_completion_tokens`·`reasoning_effort` 규칙이 달라 코드가 모델을 감지해 파라미터를 자동 전환합니다(구형 `gpt-4o` 계열은 종전대로 `temperature`+`max_tokens`).
- 모델·추론량·타임아웃 조정:
  ```bash
  SANGKWON_LLM_MODEL=gpt-5.5 SANGKWON_LLM_REASONING_EFFORT=none SANGKWON_LLM_TIMEOUT=90 python3 run.py
  ```
  - 리포트는 숫자 '번역' 작업이라 `reasoning_effort=none`(기본)으로 추론 토큰을 아껴 비용·지연을 최소화합니다. 더 저렴하게는 `SANGKWON_LLM_MODEL=gpt-5.4-mini` 등으로 교체.
- 리포트 생성 시에만 호출되므로(예측 루프 밖) API 비용이 리포트 단가 대비 미미 — 문서의 마진 논리와 일치.
